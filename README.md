## :compass: 웹사이트 개요

### :pushpin: 프로젝트 소개
기존 PlayStation 웹사이트는 정적인 레이아웃과 단조로운 인터페이스로 인해
브랜드의 역동성과 몰입감을 충분히 전달하지 못하고 있었습니다.

본 프로젝트는 이를 개선하여
**시네마틱 스크롤 인터랙션 + 데이터 기반 UI 구조 + 반응형 최적화**를 통해
웹에서도 게임과 같은 몰입 경험을 제공하는 것을 목표로 했습니다.

## :brain: 어려웠던 점 & 해결 방식

### :one: 풀페이지 다중 스와이퍼 스크롤 문제

**문제**
- 풀페이지로 잡은 다중 스와이퍼 슬라이드를 전부 스크롤하면 다음섹션으로 이동이 안됨

**해결**
- 비디오 Swiper와 텍스트 Swiper를 동기화
- GSAP ScrollTrigger를 사용하여 제어

---

### :two: 그리드 -> 스와이퍼 레이아웃 전환 

**문제**
- 단순 CSS 변경이 아니라, JS 인스턴스 생성/해제 관리가 필요

**해결**
- breakpoint 변경 시 Swiper 인스턴스를 init / destroy로 명확히 관리하여 중복 초기화 및 이벤트 누적 문제를 방지
- resize 시 destroy(true, true)로 DOM 스타일 및 이벤트를 정리하여 중복 초기화 방지

---

### :three: 반응형에서 클릭효과 중복

**문제**
- 한 번 클릭 시 이동 + 상태 변경이 동시에 발생

**해결**
- 우선 PC와 터치 디바이스의 인터랙션 특성을 분리 설계
- 모바일/태블릿에서는 2-step 클릭(활성화 → 이동) 패턴을 적용하여 오클릭을 방지
