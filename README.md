## :compass: 웹사이트 개요

### :pushpin: 프로젝트 소개
기존 PlayStation 웹사이트는 정적인 레이아웃과 단조로운 인터페이스로 인해
브랜드의 역동성과 몰입감을 충분히 전달하지 못하고 있었습니다.

본 프로젝트는 이를 개선하여
**시네마틱 스크롤 인터랙션 + 데이터 기반 UI 구조 + 반응형 최적화**를 통해
웹에서도 게임과 같은 몰입 경험을 제공하는 것을 목표로 합니다.

---

### :bust_in_silhouette: 대상 사용자

- 최신 게임 소식과 하드웨어 정보를 빠르게 확인하고 싶은 Core 게이머
- 취향 기반 게임 탐색을 원하는 잠재 고객
- 모바일 환경에서 콘텐츠를 소비하는 사용자

---

### :dart: 사용 목적

- PS5 및 PS5 Pro 기기 정보 탐색
- 게임 스토어 필터링 및 상세 페이지 확인
- 위시리스트 관리
- PlayStation 최신 뉴스 확인

---

### :sparkles: 주요 특징

- 반응형 웹 기반 (Desktop / Tablet / Mobile)
- GSAP + ScrollTrigger 기반 하이엔드 인터랙션
- URLSearchParams 기반 단일 템플릿 전략
- LocalStorage 기반 위시리스트 시스템
- 조건부 Swiper 생성/해제 구조

## :brain: 어려웠던 점 & 해결 방식

### :one: 반응형 구간별 인터랙션 분기

**문제**
PC와 Mobile에서 인터랙션 방식이 달라 동일 로직 사용 불가

**해결**
- break point 상수 정의
- resize 이벤트 기반 분기 처리
- Swiper 인스턴스 destroy / 재생성 구조 설계

---

### :two: 단일 템플릿 기반 동적 렌더링

**문제**
게임 상세 페이지를 각각 만들 경우 유지보수성 저하

**해결**
- games.js 중앙 데이터 관리
- URLSearchParams로 id 파싱
- 데이터 바인딩 함수 모듈화

---

### :three: 위시리스트 상태 유지

**문제**
페이지 이동 시 선택 정보 초기화

**해결**
- LocalStorage 기반 상태 저장
- 초기 로딩 시 저장 데이터 복원 로직 구성

---
